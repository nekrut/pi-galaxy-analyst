Skills

GitHub repos with Claude-Code-style skills (top-level AGENTS.md + nested SKILL.md files). The agent fetches them on demand via the skills_fetch tool. Alpha restriction: only repos under github.com/galaxyproject/* are accepted (the agent treats fetched skills as authoritative instructions, so the allowlist limits prompt-injection surface). galaxy-skills is the default and ships with Loom; you can disable or remove it, but a default is re-seeded if all repos are removed.

Safety

By default Loom gates the AI's local actions: shell commands and writes outside your workspace need your approval, and destructive patterns are blocked. Bypassing permissions turns that gate off entirely — the AI can run any command and edit any file as you, with no prompts. Only enable this in an environment you fully control.

Enabling this takes effect immediately (it does not wait for Save) and persists until you turn it off. A confirmation dialog appears before it's enabled.
Runs the AI's bash commands inside an OS sandbox: bash writes are confined to your analysis directory and its network is limited. Off by default because it restricts bash network access (e.g. some downloads). Takes effect after Save (restarts the session). Send feedback

Your feedback goes straight to the Loom team's private feedback store — no GitHub account needed. If that service is unreachable, Orbit falls back to opening a pre-filled issue on github.com/galaxyproject/loom with only your title and description (no diagnostics), which you submit there.
What's included?

System info: Orbit version, OS, Electron/Node versions, LLM model + provider name (no API key), Galaxy connection state (no credentials).

Logs: last 60 events of activity.jsonl (one line each: tool name, arguments, and a truncated result -- credentials and API keys redacted) and the last ~200 lines of the in-app shell stream. These can include filenames, tool arguments, and command output.

Both are on by default for the beta so we get useful diagnostics. Feedback goes to Loom's private feedback store; uncheck either box to leave it out. If the store is unreachable and we fall back to a public GitHub issue, only your text is sent — never these diagnostics.
